Challenge 1

char의 크기는 1, short의 크기는 2

int의 크기는 4, long의 크기는 Windows: 4 / Linux: 8

long long의 크기는 8, float의 크기는 4

double의 크기는 8, long double의 크기는 Windows: 8 / Linux: 16

bool의 크기는 1



long과 long double의 크기가 Windows와 Linux에 따라 달라지는 이유는

운영체제와 컴파일러의 데이터 모델에 따라 자료형의 크기가 달라지기 때문입니다.



signed와 unsigned는 비트를 해석하는 방식만 다르기 때문에 크기 차이는 없습니다.



Challenge 3

signed int ( -2^31 ~ 2^31 - 1)

INT\_MAX: 2,147,483,647

INT\_MAX + 1: -2,147,483,648



unsigned int (0 ~ 2^32 - 1)

INT\_MAX: 4,294,967,295

INT\_MAX + 1: 0



signed int(부호 있는 int)는 2의 보수 방식으로 표현합니다.

따라서 최댓값인 01111111 11111111 11111111 11111111(2)에 1을 더하면

10000000 00000000 00000000 00000000(2)가 되어 MSB가 1이 되면서 음수로 해석됩니다.



unsigned int(부호 없는 int)는 부호가 없이 0 이상의 값만 표현합니다.

따라서 최댓값인 11111111 11111111 11111111 11111111(2)에 1을 더하면

1 00000000 00000000 00000000 00000000(2)이 되고 int는 32bits만 저장하므로 맨 앞에 올림수 1은 버려지고 32개의 0만 남아 0이 됩니다.



Challenge 4

int 변수를 문자열 형식지정자인 %s로 각각 출력하려 하였지만

프로그램이 비정상적으로 종료가 되었습니다.

이렇게 되는 이유는 %s는 문자열을 출력하라는 의미로 넘어오는 인자를 메모리 주소로 해석합니다.

그런데 정수 변수를 넣게 되면 그 정수 변수가 그대로 주소값으로 해석되어 그 주소로 넘어가는데

그 주소에 있는 값이 유효한 문자열이 아니거나 접근할 수 없는 영역일 수 있기에 프로그램이 강제 종료되는 것입니다.



포맷 스트링 취약점은 프로그램이 사용자 입력 데이터를 printf와 같은 형식화된 출력 함수의 포맷 문자열 인자로 직접 사용할 때 발생합니다.

%x 등의 형식지정자를 입력하여 메모리를 읽거나 %n 등의 형식지정자로 임의의 주소에 데이터를 쓰는 등의 공격을 할 수 있습니다.
